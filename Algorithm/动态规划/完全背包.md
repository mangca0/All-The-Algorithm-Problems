与 01 背包的区别仅在于 **每种物品可以选择无限次**。

完全背包：类似于 01 背包。有 $n$ 种物品，第 $i$ 种物品的价值为 $values[i]$，体积为 $costs[i]$，每种物品无限重复次选，$\begin{cases}至多装 capacity，求方案数/最大价值和。 \\恰好装 capacity，求方案数/最大/最小价值和。 \\至少装 capacity，求方案数/最小价值和。\end{cases}$ 

实现时，常用 **选或不选** 的思路。

时间复杂度一般都是 $O(物品个数 \cdot 背包容量)$。

## 模板
状态转移方程与 01 背包类似，**选** 策略时，$i$ 参数不变：$$dfs(i, j) = max(dfs(i - 1, j) + dfs(i, s - costs[i]) + value[i])$$

### 恰好型 求最小价值和
记忆化搜索：
```python
@cache
def dfs(i: int, j: int) -> int:
	if i < 0:
		return 0 if j == 0 else inf
	if j < coins[i]:
		return dfs(i - 1, j)
	return min(dfs(i - 1, j), dfs(i, j - coins[i]) + 1)
ans = dfs(len(coins) - 1, amount)
return -1 if ans == inf else ans
```

一维空间优化：
```python
f = [inf] * (amount + 1)
f[0] = 0
for x in coins:
	for j in range(x, amount + 1):
		f[j] = min(f[j], f[j - x] + 1)
ans = f[-1]
return -1 if ans == inf else ans
```

这题是 [力扣 322. 零钱兑换](https://leetcode.cn/problems/coin-change/)。

### 恰好型 求方案数
记忆化搜索：
```python
@cache
def dfs(i: int, j: int) -> int:
	if i < 0:
		return 1 if j == 0 else 0
	if j < coins[i]:
		return dfs(i - 1, j)
	return dfs(i - 1, j) + dfs(i, j - coins[i])
ans = dfs(len(coins) - 1, amount)
return ans
```

一维空间优化：
```python
f = [0] * (amount + 1)
f[0] = 1
for x in coins:
	for j in range(x, amount + 1):
		f[j] += f[j - x]
return f[-1]
```

这题是 [力扣 518. 零钱兑换 II](https://leetcode.cn/problems/coin-change-ii/)。
