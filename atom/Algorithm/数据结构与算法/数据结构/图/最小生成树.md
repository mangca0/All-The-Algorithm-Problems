# Minimum Cost Spanning Tree

针对**无向带权图**，在保证**连通性**的情况下，**总边权**最小。

可能不止一颗。

n个点一共n-1条边  

一定是最小瓶颈树。

## Kruskal
最常见并且好写的最小生成树算法，基本思想是从小到大加入边，是种贪心算法。
### 步骤
1. 把边按边权以升序排序。
2. 枚举每条边 $(u, v, w)$
	1. 如果 $u$ 和 $v$ 不在同一个集合里，则选这条边。
	2. 否则，跳过。
3. 选够 $n-1$ 条，或遍历完所有边，结束。

需要相应的数据结构来支持：
- 具体来说，维护一个森林，查询两个结点是否在同一棵树中，连接两棵树。
- 抽象一点地说，我们需要维护一堆 **集合**，查询两个元素是否属于同一集合，合并两个集合。

这可以用 **并查集** 实现，同时也不需要建图。
```java
// 以下是并查集操作
public static void build() {
	for (int i = 0; i < n; i++) {
		fa[i] = i;
	}
}

public static int find(int i) {
	return fa[x] = (fa[x] == x ? x : find(fa[x])); // 路径压缩优化
}

// 返回是否合并成功
public static boolean merge(int x, int y) {
	x = find(x);
	y = find(y);
	if (x == y) {
		return false;
	} else { // 这里可以根据树的规模，用启发式合并优化
		fa[x] = y;
		return true;
	}
}

// 主程序
public static void Kruskal {
	Arrays.sort(edges, (a, b) -> a[2] - b[2]); 
	// Arrays.sort(edges, Comparator.comparingInt(e -> e[2])); // 这样写也行
	int ans = 0;
	int edgeCnt = 0;
	for (int[] edge : edges) {
		if (merge(edge[0], edge[1])) {
			edgeCnt++;
			ans += edge[2];
		}
	}
	System.out.println(edgeCnt == n - 1 ? ans : "No");
}
```
**时间复杂度：**这里有两个点：
1. 不考虑路径压缩，单次 ‎`find` 最坏是树高 $h$，复杂度 $O(h)$，最坏情况下 $h$ 可以接近 $n$。
2. 加上**路径压缩**（还可以再配合按秩合并/按大小合并），有结论：  
    单次 ‎`find` 的**均摊复杂度**是 ‎$O(α(n))$，$α$ 其中 是反阿克曼函数，增长极其缓慢。

所以单次 ‎`find`：均摊 $O(α(n)) ≈ O(1)$。
同理单次 `merge`：$O(α(n)) + O(α(n)) + O(1) ≈ O(α(n)) ≈ O(1)$（均摊）。

总体时间复杂度：$O(m\log m+n+m)=O(m\log m)$。瓶颈在于排序。

## Prim
Prim 算法是另一种常见并且好写的最小生成树算法，该算法的基本思想是从一个结点开始，不断加点（而不是 Kruskal 算法的加边）。是一种经典算法。

1. 解锁的点集set、解锁的边集heap（小），初始化为空。
2. 可从任意边开始，开始点加入set，开始边加入heap。
3. heap维护权值，弹出e，看看e去往的点v
	1. 如果v已经在set里，continue
	2. 如果不在，要，把v的出边加进heap
4. 重复3直到heap为空

$O(mlogm+n+m)$
