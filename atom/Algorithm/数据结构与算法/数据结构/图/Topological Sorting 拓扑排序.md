## 定义
拓扑排序（Topological sorting）解决的是：一个有向无环图的所有节点排序。
把拓扑排序想象成一个黑盒，给它一堆杂乱的先修课约束，它会给你一个井井有条的课程学习安排。
因为要求无环，从而保证每条边都是从排在前面的点，指向排在后面的点，也就是每个节点的前置节点都在这个节点之前。

我们可以拿大学排课的例子来描述这个过程，教务处安排这些课程，使得在逻辑关系符合的情况下排出课表，就是拓扑排序的过程。

但我们发现：如果想要学习 数据结构，还得先学 操作系统，而 操作系统 的前置课程又是 数据结构，那么到底应该先学哪一个（不考虑同时学习的情况）？在这里，数据结构 和 操作系统 间就出现了一个环，显然同学们现在没办法弄清楚自己需要先学什么了，也就没办法进行拓扑排序了。如果有向图中存在环路，那么就没办法进行拓扑排序。
拓扑排序的目标是将所有节点排序，使得排在前面的节点不能依赖于排在后面的节点。

入度删除法。删除的点和总的点不相等=拓扑排序进行不下去，有环。
## AOV 网
**在一个表示工程的有向图中，用顶点表示活动，用弧表示活动之间的优先关系，这样的有向图为顶点表示活动的网，我们称为 AOV 网（Activity On Vertex Network）。** AOV 网中的弧表示活动之前存在的某种制约关系。

所谓**拓扑排序，其实就是对一个有向图构造拓扑序列的过程**。构造时会有两个结果，如果此网中的每个顶点都被输出，则说明他是不存在环（回路）的 AOV 网；反之，如果输出的顶点少了，则说明这个网存在环（回路），可以用该性质判断有向图是否有环。

一个 AOV 网的拓扑序列和拓扑排序的顺序可能不唯一。

## 算法实现
1. 从图中选择一个入度为零的点。
2. 输出该顶点，从图中删除此顶点及其所有的出边。

重复上面两步，直到所有顶点都输出，拓扑排序完成，或者图中不存在入度为零的点，此时说明图是有环图，拓扑排序无法完成。

```java
int[] indegree = new int[n]; // 入度表 在建图过程中构建即可 建图过程省略

boolean topoSort() {
	// 这里用数组模拟 队列，因为入度为 0 的点之间不在乎先后关系，所以这里用 栈 实现也可以
	// 用数组模拟结束后，队列可以直接当成拓扑序列，比较方便
	int[] queue = new int[n]; 
	int l = 0, r = 0;
	for (int i = 0; i < n; i++) {
		if (indegree[i] == 0) {
			queue[l++] = i;
		}
	}
	while(l < r) {
		int u = queue[l++];
		for (int v : g[u]) { // 这里假设用邻接表建图 ArrayList<Integer>[] g
			if (--indegree[v] == 0) {
				queue[r++] = v;
			}
		}
	}
	// 如果构建成功，queue 就是拓扑序列，可以打印或直接返回
	return r == n ? true : false; // 用 l 也可以，因为此时 l=r
}
```
时间复杂度：$O(E+V)$。
空间复杂度：$O(V)$。

### 应用
拓扑排序可以判断图中是否有环，还可以用来判断图是否是一条链。拓扑排序可以用来求 AOE 网中的关键路径，估算工程完成的最短时间。

### 求字典序最大/最小的拓扑排序
将算法中的队列替换成最大堆/最小堆实现的优先队列即可，此时总的时间复杂度为 $O(E+VlogV)$。
